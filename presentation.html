01

Contexte du Projet

Pourquoi AskNova AI ?

Le Constat

  • Les interfaces vocales IA existantes (Vapi, etc.) nécessitent un numéro de téléphone
  • La latence des solutions cloud dépasse souvent 3-5 secondes
  • Pas de solution web accessible directement depuis le navigateur
  • Manque de naturalité dans les conversations IA vocales

L'Objectif

  • Créer une expérience vocale comparable à une conversation humaine
  • Latence totale < 2 secondes (parole → réponse audio)
  • Communication full-duplex avec interruptions naturelles (barge-in)
  • Architecture micro-services scalable et conteneurisée

💡 Vision : Une plateforme web où l'utilisateur parle à une IA aussi naturellement qu'à un humain — en un clic, sans installation, sans numéro de téléphone.

Deep Dive — Service CAG (Cerveau)

Llama 3.2-3B — Inférence Optimisée

Optimisations KV-Cache

  • Pré-computation : La base de connaissances est encodée en KV-cache au démarrage
  • Troncature : Après chaque requête, le cache est tronqué à la longueur knowledge-base
  • Persistance : Cache sauvegardé sur disque pour démarrage instantané
  • Résultat : TTFT ~80-120ms au lieu de ~500ms+

Mémoire Conversationnelle

  • Historique des 10 derniers tours
  • Profil utilisateur (prénom, préférences)
  • Extraction automatique du nom
  • Prompt système compressé (80 tokens)

Quantification 4-bit

  • Modèle 3B paramètres en seulement ~2.5 Go VRAM
  • BitsAndBytes NF4 quantization
  • Flash Attention 2 (fallback eager)
  • TF32 activé pour accélération matmul
  • Décodage greedy (déterministe + rapide)

Deep Dive — Gateway Orchestrateur

6 Workers Concurrents par Client

🎤 STT Loop

Connexion WebSocket vers le service STT. Reçoit les segments transcrits et les mots en temps réel. Gère le barge-in.

🧠 CAG Loop

Envoie les requêtes au LLM. Reçoit les tokens en streaming. Accumulation tonale intelligente.

🗣️ Synth Worker

Convertit les chunks textuels en audio via Azure TTS. SSML avec prosodie adaptée au ton.

🔊 Play Worker

Stream les frames PCM vers le client. Notifie le STT quand l'IA parle (pour echo gate).

💓 Heartbeat

Ping/pong keep-alive pour maintenir la connexion WebSocket ouverte.

⏰ Idle Watchdog

Reset automatique après 120s d'inactivité. Libère les ressources GPU.

Difficultés Rencontrées

Défis Techniques & Solutions

🔄 Écho de la Voix IA

Problème : Le microphone captait la voix de l'IA jouée par les haut-parleurs, créant une boucle de rétroaction — l'IA se « parlait à elle-même ».

Solution : Développement d'un système à 3 couches : gate temporelle (AEC), empreinte spectrale (Voice Gate), et filtre textuel. La combinaison élimine >99% des échos.

⏱️ Latence du Pipeline

Problème : Le pipeline séquentiel (STT → LLM → TTS) cumulait les latences, dépassant 4-5 secondes initialement.

Solution : Streaming à chaque étape + KV-cache precompute + tonal chunking. Le premier chunk audio arrive en ~1.2s au lieu de ~5s.

🧠 Contrainte VRAM (6 Go)

Problème : Le modèle LLM + Whisper + VAD doivent coexister sur un GPU RTX 4050 avec seulement 6 Go de VRAM.

Solution : Quantification NF4 (3B → 2.5 Go), CTranslate2 pour Whisper (optimisé), Silero VAD CPU-only. Total < 5 Go VRAM.

🎤 Mots Perdus / Dupliqués (STT)

Problème : Le streaming Whisper perdait des mots en fin de fenêtre ou les dupliquait entre les passes.

Solution : Algorithme « v2.0 » avec cursor tracking (plus-long-préfixe-commun), live pass toutes les 400ms, et flush pass avec beam=5 au silence.

🔗 Autres Défis

  • Barge-in : Distinguer l'écho IA d'une vraie interruption utilisateur
  • Hallucinations LLM : Répétition de phrases → Repetition Guard
  • Connexion WebSocket : Reconnexion automatique + backoff exponentiel
  • Prosodie TTS : Détection automatique du ton pour SSML naturel
05

Difficultés Rencontrées

Leçons Apprises

💡 Streaming Partout

La latence perçue dépend du premier byte, pas de la réponse complète. Streamer chaque étape du pipeline réduit la latence perçue de 70%.

💡 Défense en Profondeur

Un seul filtre anti-écho ne suffit jamais. La combinaison timing + acoustique + textuel est nécessaire pour une robustesse >99%.

💡 KV-Cache = Game Changer

Le pré-calcul du KV-cache pour le contexte statique (knowledge base) réduit le TTFT de 500ms → 80ms. Technique sous-estimée mais critique.

💡 Quantification Well-Tuned

La quantification 4-bit avec NF4 + Flash Attention 2 conserve ~95% de la qualité du modèle tout en divisant la VRAM par 4.

💡 Micro-services = Agilité

Chaque service peut être redémarré, mis à jour ou débogué indépendamment. Essentiel pour un pipeline aussi complexe.

💡 Test Tôt, Test Souvent

Le manque initial de tests a rendu le débogage du pipeline difficile. Les tests CRUD rétrospectifs ont révélé plusieurs bugs cachés.
